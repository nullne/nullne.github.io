知识储备

  • encode

    编码方式这个东西每次出问题了才去搜搜，下一次又遇到又出错。所以这次我索性把他弄个明白。
    首先理清楚这两个概念:
    • 系统编码方式
    • 代码内部编码方式

*nix平台上输入 locale 即可以得到其系统编码方式，在代码（python）上面标注# -*- coding:UTF-8 -*-即标明代码内部编码方式为UTF-8.就相当于两个世界的语言一样，彼此是无法沟通的。这时候就出现了一个大家都懂的公共语言--unicode,他们之间的翻译是基于这个编码方式的 看下面这个例子: 

In [1]: unicode_str = u"吻得太逼真"
In [2]: unicode_str
Out[2]: u'\u543b\u5f97\u592a\u903c\u771f'
In [3]: utf8_str = unicode_str.encode('UTF-8')
In [4]: utf8_str
Out[4]: '\xe5\x90\xbb\xe5\xbe\x97\xe5\xa4\xaa\xe9\x80\xbc\xe7\x9c\x9f'
In [5]: gbk_str = unicode_str.encode('GBK')
In [6]: gbk_str
Out[6]: '\xce\xc7\xb5\xc3\xcc\xab\xb1\xc6\xd5\xe6'
In [7]: print utf8_str
Out[7]: 吻得太逼真
In [8]: print gbk_str
Out[8]: �ǵ�����

很容易就能明白对不？utf8_str和gbk_str都是基于unicode_str编码的，直接输变量名得到的OUT是repr，print得到的是str。简单的说repr显示的是正式字符表达式串，而str采用非正式字符串表达式。最后一行出现乱码可以这么理解，系统编码方式是UTF-8，他得到一串以GBK编码的字符时，当然就不明白如何正确显示，所以就乱码掉了。 再看一个代码中的例子： 

comment = raw_input("COMMENT: ")
comment = comment.decode(sys_encode)  

vote = requests.post('http://10.3.240.70/jxpgXsAction.do?oper=wjpg', data=post_data, cookies=login.cookies, headers=headers)
out =  vote.content
if str(out).find('成功') >= 0:
    print "---- success on " + i['bprm']
else:
    print "---- fail on " + i['bprm']

代码编码方式是GBK, 所以“成功”字样的编码方式就是GBK,网页返回的也是GBK，所以在搜索的时候就不用做编码转换了。这就是为什么我要把代码的编码方式改为GBK。但是因为系统的编码方式是UTF-8，所以在和系统做交互的时候一定要注意编码的转换，这样才不会错。
所以总结哈，然是牵扯到两个环境中的文本交互，就一定需要注意编码方式的问题，比如说刚才提到的系统编码和代码自身编码不同，还有网页编码和代码编码。值得一提的是不能直接把unicode编码的东西直接POST给网页，这样也可能会出错的,因为unicode本身也只是一种编码方式
这篇写的非常专业，如果需要的话可以过去爽爽--> 字符编码详解
